<task type="auto">
  <name>Task 1: Convention Registry schema and registry builder</name>
  <files>src/core/schema.ts, src/core/registry.ts, test/core/schema.test.ts</files>
  <action>
    Create src/core/schema.ts with Zod v4 schemas. Use `import { z } from "zod"` (v4 import path is the same as v3).

    Define these schemas (all at module top-level, never inline):
    - ConventionCategorySchema: z.enum with categories: stack, naming, architecture, error_handling, testing, imports, other
    - EvidenceRefSchema: { file: string, line: number | null }
    - ConventionEntrySchema: { id: string (uuid), category, pattern: string (min 1), confidence: number (0-1), evidence: EvidenceRef[], metadata?: Record<string, unknown> }
    - StackInfoSchema: { language: string, framework?: string, testRunner?: string, buildTool?: string, packageManager?: string, nodeVersion?: string }
    - ArchitectureInfoSchema: { pattern?: string, layers: string[], entryPoints?: string[] }
    - ConventionRegistrySchema: { version: literal "1", projectPath: string, generatedAt: string (datetime), stack: StackInfo, conventions: ConventionEntry[], architecture: ArchitectureInfo, metadata?: Record<string, unknown> }

    Export all schemas AND their inferred types via z.infer<>. Do NOT write separate TypeScript interfaces.

    IMPORTANT Zod v4 notes:
    - Use `z.record(z.string(), z.unknown())` (two args required in v4)
    - Error shape uses `.issues` not `.errors`
    - If uuid validation is needed at top level, check if z.uuid() exists; if not, use z.string().uuid() (v4 may have moved this)
    - Use `.nullable()` not `.nullish()` for line numbers

    Create src/core/registry.ts:
    - `createRegistry(projectPath: string)`: Factory that returns a new empty ConventionRegistry with version "1", current datetime, empty conventions/layers
    - `addConvention(registry: ConventionRegistry, entry: Omit<ConventionEntry, "id">)`: Adds a convention with auto-generated UUID (use crypto.randomUUID())
    - Both functions should validate via the Zod schema (safeParse on the result)

    Create test/core/schema.test.ts:
    - Test: valid minimal registry passes validation
    - Test: valid registry with conventions passes validation
    - Test: invalid confidence (> 1) is rejected
    - Test: invalid confidence (< 0) is rejected
    - Test: missing required fields are rejected
    - Test: inferred types match expected shape (compile-time check via satisfies)
    - Test: createRegistry produces a valid registry
    - Test: addConvention adds entry with generated UUID

    Use explicit Vitest imports: `import { describe, it, expect } from "vitest"`
    Use .js extensions in all relative imports: `from "../../src/core/schema.js"`

    Run `npm run test` after creating tests. All must pass.
  </action>
  <verify>
    - `npm run test -- test/core/schema.test.ts` passes all tests
    - `npm run typecheck` passes
  </verify>
  <done>Convention Registry schema validates sample data, rejects invalid data. Registry builder creates valid registries with auto-generated UUIDs. All types are z.infer<> derived. Tests prove it.</done>
