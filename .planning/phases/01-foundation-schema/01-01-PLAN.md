  <files>package.json, .gitignore</files>
  <action>
    Initialize the project with `npm init -y` in the repo root. Then configure package.json:

    - Set `"name": "@ez-corp/ez-context"`
    - Set `"type": "module"` (ESM-only)
    - Set `"version": "0.0.0"`
    - Set `"license": "ISC"`
    - Set `"engines": { "node": ">=20.19.0" }`
    - Set `"main": "./dist/index.js"`
    - Set `"exports": { ".": { "import": "./dist/index.js", "types": "./dist/index.d.ts" } }`
    - Add scripts:
      - `"build": "tsdown"`
      - `"dev": "tsx src/cli.ts"`
      - `"test": "vitest run"`
      - `"test:watch": "vitest"`
      - `"lint": "eslint src/ test/"`
      - `"typecheck": "tsc --noEmit -p tsconfig.test.json"`

    Install runtime dependencies:
    ```
    npm install zod globby @ez-corp/ez-search
    ```

    Install dev dependencies:
    ```
    npm install -D tsdown tsx vitest typescript @types/node eslint @eslint/js typescript-eslint
    ```

    Create .gitignore with: node_modules, dist, coverage, .ez-search, *.tsbuildinfo

    IMPORTANT: If @ez-corp/ez-search fails to install (not published yet), install without it and add it to package.json dependencies manually as `"@ez-corp/ez-search": "*"`. The bridge will adapt to the real API later. Do NOT block scaffolding on this.
  </action>
  <verify>
