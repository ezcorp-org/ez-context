Create `src/emitters/writer.ts` with:
- Export constants `MARKER_START = "<!-- ez-context:start -->"` and `MARKER_END = "<!-- ez-context:end -->"`
- Export `writeWithMarkers(filePath: string, content: string): Promise<void>`
- Three paths:

  3. File exists with markers: splice -- `before + MARKER_START\n{content}\n{MARKER_END} + after`
- Use `node:fs/promises` readFile/writeFile and `node:fs` existsSync
- Handle edge case: if only one marker is present (start but no end, or vice versa), treat as "no markers" and append

Follow the exact pattern from RESEARCH.md Pattern 2 but add the single-marker-present guard.
