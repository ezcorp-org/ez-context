  <action>
Create `src/emitters/claude-md.ts`:
- Import `Eta` from `"eta"`, instantiate with `new Eta({ autoEscape: false })` (CRITICAL: autoEscape false for markdown)
- Import `ConventionRegistry`, `ConventionEntry` from `../core/schema.js`
- Export `renderClaudeMd(registry: ConventionRegistry, confidenceThreshold: number): string`
- Build a data-prep function that filters conventions to `confidence >= confidenceThreshold`, groups by category, and computes boolean flags for section visibility (hasStack, hasConventions, hasArchitecture, hasNaming, hasTesting, hasImports)
- Eta template string for CLAUDE.md structure:
  ```
  # Project Context

  ## Stack  (only if stack.language !== "unknown")
  - Language: {language}
  - Framework: {framework}  (only if present)
  - Build: {buildTool}  (only if present)
  - Package Manager: {packageManager}  (only if present)
  - Test Runner: {testRunner}  (only if present)

  ## Conventions  (only if has conventions after filtering)
  Group by category, render each as:
  - **{category}**: {pattern} (one line per convention)

  ## Architecture  (only if architecture.pattern or layers.length > 0)
  - Pattern: {architecture.pattern}  (if present)
  - Layers: comma-separated list  (if layers.length > 0)
  ```
- Use `<%` for control flow, `<%= it.value %>` for output (safe with autoEscape: false)
- Assert renderString result is non-null (see RESEARCH.md Pitfall 2)
- Guard every section: if no data, emit nothing (no heading, no empty bullet)
- Keep output under 100 lines for a typical project

Create `src/emitters/agents-md.ts`:
- Same Eta setup (autoEscape: false)
- Import `ConventionRegistry` from `../core/schema.js`
- Export `renderAgentsMd(registry: ConventionRegistry, confidenceThreshold: number): string`
- Data-prep: extract commands from conventions with `metadata.command` and `metadata.scriptName`, filter to confidence >= threshold
- Eta template for AGENTS.md following Linux Foundation 6-area structure:
  ```
  # AGENTS.md

  ## Commands  (only if has commands)
  Render each as: - `{scriptName}`: `{command}`
  Include build, test, lint, typecheck, format if detected

  ## Testing  (only if has testing conventions)
  - Test runner: {testRunner}
  - Test patterns from conventions with category=testing

  ## Project Structure  (only if architecture has layers or pattern)
  - Architecture pattern if detected
  - Key layers

  ## Code Style  (only if has naming or imports conventions)
  - Naming conventions (casing rules)
  - Import patterns

  ## Git Workflow  (only if has git-related conventions -- may be empty for many projects)

  ## Boundaries
  - Static section: "Do not modify auto-generated sections between ez-context markers"
  ```
- Same guards: no heading if no content for that section
- Target under 80 lines output for a typical project
- Do NOT restate package.json dependencies, tsconfig compiler options, or anything agents read from config files directly (GEN-07)
  </action>
  <verify>
`bun run build` succeeds. `bun run lint` passes. Manually inspect: importing renderClaudeMd and calling it with a sample registry produces reasonable markdown output.
  </verify>
  <done>
