  <files>

  <action>
    Refine `src/commands/generate.ts`:
    - When `--yes` is passed OR stdout is not a TTY (`!process.stdout.isTTY`): use ora with `isSilent: false` but ensure no interactive prompts are used. ora already handles non-TTY gracefully (falls back to plain text), so --yes mainly signals "don't prompt for confirmation".
    - Add spinner text updates during the pipeline:
      1. Start: "Analyzing project conventions..."
      2. After extractConventions succeeds: spinner.succeed with convention count, e.g. "Found {N} conventions"
      3. Before emit: new spinner "Generating context files..."
      4. After emit succeed/dry-run: spinner.succeed with file count or "Dry run complete"
    - Improve dry-run output: print a boxed header "DRY RUN -- no files will be written", then for each file (CLAUDE.md, AGENTS.md) print a separator line with the filename, then first 20 lines of content followed by "... ({total} lines)" if content exceeds 20 lines. This prevents flooding the terminal.
    - Improve normal output: after writing, print a summary box:
      ```
      Generated 2 files:
        CLAUDE.md  (path)
        AGENTS.md  (path)
      ```
    - CLI-08 requirement: "Clear terminal output showing what was generated and where" -- ensure every generate run ends with a clear summary of files written and their absolute paths.

    Refine `src/commands/inspect.ts`:
    - Same spinner pattern: start -> succeed with count
    - Add a summary footer: "Found {N} conventions across {M} categories (threshold: {T})"
    - When `--threshold` filters out all conventions, show: "No conventions found above {threshold} confidence threshold"

    IMPORTANT: Do NOT add any interactive prompts or confirmation dialogs. The --yes flag exists to signal CI/non-interactive intent, but the current commands don't have any prompts to skip. Just ensure the flag is accepted and the code path works cleanly in non-TTY environments.
