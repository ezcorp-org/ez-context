<task type="auto">
  <name>Task 2: Implement generate and inspect command handlers</name>
  <files>
    src/commands/generate.ts
    src/commands/inspect.ts
  </files>
  <action>
    Create `src/commands/generate.ts` exporting `generateAction`:
    - Signature: `async function generateAction(pathArg: string, options: { dryRun?: boolean; yes?: boolean; output?: string; threshold?: string })`
    - Resolve `pathArg` to absolute with `path.resolve(pathArg)`
    - Import `ora` from `ora`, `chalk` from `chalk`
    - Import `extractConventions` from `../core/pipeline.js`
    - Import `emit` from `../emitters/index.js`
    - Import `type EmitOptions` from `../emitters/types.js`
    - Start spinner: `ora("Analyzing project conventions...").start()`
    - Call `const registry = await extractConventions(projectPath)`
    - Update spinner: `spinner.succeed(...)` with convention count
    - Parse threshold: `parseFloat(options.threshold ?? "0.7")`
    - Resolve outputDir: `path.resolve(options.output ?? ".")`
    - Call `const result = await emit(registry, { outputDir, confidenceThreshold, dryRun: options.dryRun ?? false })`
    - If dryRun: print `chalk.bold("\nDry run -- no files written\n")`, then print rendered content for CLAUDE.md and AGENTS.md with chalk.cyan headers
    - If not dryRun: print `chalk.green("\nGenerated:")` followed by each file path from `result.filesWritten`
    - Wrap entire body in try/catch: on error, `spinner.fail("Analysis failed")`, then `console.error(chalk.red(err.message))`, then `process.exit(1)`

    Create `src/commands/inspect.ts` exporting `inspectAction`:
    - Signature: `async function inspectAction(pathArg: string, options: { threshold?: string })`
    - Resolve path, start spinner, call `extractConventions`
    - After extraction, group conventions by category using a Map
    - Filter by threshold (parseFloat options.threshold ?? "0.7")
    - For each category group, print:
      - Category header: `chalk.bold(category.toUpperCase())`
      - Each convention: confidence indicator dot (green >= 0.8, yellow >= 0.6, red < 0.6) + pattern + gray confidence percentage
    - If no conventions found, print a yellow message saying so
    - Print summary line at end: total conventions found, categories represented
    - Same try/catch error handling pattern as generate

    IMPORTANT: Import directly from submodules, NOT from `../index.js`.
  </action>
  <verify>
    - `bun run build` succeeds with no type errors
    - `bun run typecheck` passes
    - `bun run lint` passes
    - `node dist/cli.js generate --dry-run .` runs extraction and prints rendered markdown without writing files
    - `node dist/cli.js inspect .` runs extraction and prints conventions grouped by category
  </verify>
  <done>
    `ez-context generate` analyzes the current project and writes CLAUDE.md + AGENTS.md (or previews in dry-run mode).
    `ez-context inspect` displays conventions grouped by category with confidence indicators.
    Both commands handle errors gracefully with colored output.
  </done>
</task>
