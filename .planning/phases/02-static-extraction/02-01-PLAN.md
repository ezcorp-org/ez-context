<task type="auto">
  <name>Task 2: Implement static config-file extractors</name>
  <files>
    src/extractors/static/package-json.ts
    src/extractors/static/lockfile.ts
    src/extractors/static/tsconfig.ts
    src/extractors/static/go-mod.ts
    src/extractors/static/cargo-toml.ts

  <action>
    All extractors follow the same pattern from RESEARCH.md:
    - File-existence guard: try `access(path)`, catch and return `[]`
    - Parse the file content
    - Return `Omit<ConventionEntry, "id">[]` with category, pattern, confidence, evidence, metadata
    - Never throw -- return empty array if file missing or unparseable

    **package-json.ts** (`packageJsonExtractor`):
    - Read and JSON.parse package.json
    - Detect language: TypeScript if `typescript` or `@types/node` in deps/devDeps, else JavaScript
    - Detect framework: Check deps against FRAMEWORK_MAP (react, vue, angular, next, nuxt, svelte, hono, express, fastify, koa). Use first match. Category "stack".
    - Detect test runner: Check deps against TEST_RUNNER_MAP (vitest, jest, mocha, jasmine, ava). Category "testing".
    - Extract build/test/lint scripts: For script names "build", "test", "lint", and names starting with "test:", create entries with the command. Category "stack" for build/lint, "testing" for test scripts.
    - Confidence: 0.95 for language, 1.0 for framework (it's in deps), 0.95 for test runner, 1.0 for scripts.
    - Evidence: `[{ file: "package.json", line: null }]`
    - Store useful metadata: `{ language }`, `{ framework, version }`, `{ testRunner }`, `{ scriptName, command }`

    **lockfile.ts** (`lockfileExtractor`):
    - Check lockfiles in priority order: bun.lock, bun.lockb, pnpm-lock.yaml, yarn.lock, package-lock.json
    - First found determines package manager
    - Return single entry: category "stack", pattern "Package manager: {manager}", confidence 1.0, metadata `{ packageManager: manager }`
    - Evidence: `[{ file: lockfileName, line: null }]`

    **tsconfig.ts** (`tsconfigExtractor`):
    - Read and JSON.parse tsconfig.json (handle trailing commas with a strip-comments approach or just try/catch JSON.parse)
    - Detect strict mode: `compilerOptions.strict === true`
    - Detect notable settings: target, module, moduleResolution, paths (alias usage)
    - Return entries: category "stack", e.g. "TypeScript strict mode enabled", confidence 1.0 if strict is true
    - If paths exist, add an entry: category "imports", pattern "Uses TypeScript path aliases", confidence 1.0, metadata with the alias patterns
    - Evidence: `[{ file: "tsconfig.json", line: null }]`

    **go-mod.ts** (`goModExtractor`):
    - Read go.mod as text, parse line-by-line with regex (NOT with TOML parser)
    - Extract module name from `module <path>` line
    - Extract Go version from `go <version>` line
    - Count dependencies in `require ( ... )` blocks
    - Return entry: category "stack", pattern "Go project ({moduleName})", confidence 1.0
    - Metadata: `{ language: "Go", moduleName, goVersion, dependencyCount }`

    **cargo-toml.ts** (`cargoTomlExtractor`):
    - Use `import { parse as parseToml } from "smol-toml"` to parse Cargo.toml
    - Extract package name from `[package].name`
    - Count dependencies from `[dependencies]` keys
    - Return entry: category "stack", pattern "Rust project ({packageName})", confidence 1.0
    - Metadata: `{ language: "Rust", packageName, dependencyCount }`

    IMPORTANT: Each extractor file must import types from `../types.js` (with .js extension for ESM).
  </action>
