  <name>Task 1: Pipeline module with StackInfo population and deduplication</name>
  <files>
    src/core/pipeline.ts
    src/index.ts
  </files>
  <action>
    **pipeline.ts** -- Create `extractConventions(projectPath: string, options?: ExtractorOptions): Promise<ConventionRegistry>`:

    1. Import all 9 extractors:
       - packageJsonExtractor, lockfileExtractor, tsconfigExtractor, goModExtractor, cargoTomlExtractor (static)
       - ciExtractor, projectStructureExtractor (static)
       - namingExtractor, importsExtractor (code)

    2. Create an ExtractionContext from projectPath + options

    3. Create an empty registry via `createRegistry(projectPath)`

    4. Call `runExtractors(allExtractors, ctx, registry)` to get populated registry

    5. **Deduplicate conventions** by category+pattern:
       - Group conventions by `${category}:${pattern}`
       - For duplicates, keep the one with higher confidence
       - Merge evidence arrays from duplicates (deduplicate evidence by file+line)

    6. **Populate StackInfo** from extracted conventions (post-extraction pass):
       - Scan conventions with category "stack" for metadata keys:
         - `metadata.language` -> `stack.language`
         - `metadata.framework` -> `stack.framework`
         - `metadata.testRunner` -> `stack.testRunner`
         - `metadata.packageManager` -> `stack.packageManager`
         - `metadata.buildTool` -> `stack.buildTool`
       - Scan conventions with category "testing" for `metadata.testRunner` -> `stack.testRunner`
       - For `stack.buildTool`: check scripts metadata for "build" script command
       - First match wins for each field (extractors ordered by confidence)

    7. Return the final registry with populated stack and deduplicated conventions. Validate with `ConventionRegistrySchema.parse()` before returning.

    **src/index.ts** -- Update barrel exports:
    - Add: `export { extractConventions } from "./core/pipeline.js"`
    - Add: `export type { Extractor, ExtractionContext, ExtractorOptions } from "./extractors/types.js"`
    - Add: `export { runExtractors } from "./extractors/index.js"`
    - Keep all existing exports
  </action>

  </verify>
