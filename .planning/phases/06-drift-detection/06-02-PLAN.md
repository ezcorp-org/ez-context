**claim-scorer.ts:**

1. Export types:
```typescript
export type ClaimStatus = "GREEN" | "YELLOW" | "RED";

export interface ScoredClaim {
  claim: Claim;
  status: ClaimStatus;
  score: number;           // Top bridge.search() score (0.0-1.0)
  evidence: SearchResult[]; // Top k results
}
```

2. Export `scoreClaims(claims, bridge, onProgress?)` that:
   - Batches claims in groups of 10 (BATCH_SIZE constant)
   - For each batch, runs `Promise.all(batch.map(claim => scoreSingleClaim(claim, bridge)))`
   - `scoreSingleClaim`: calls `bridge.search(claim.text, { k: 5 })`, takes top score (or 0 if no results), classifies: GREEN >= 0.65, YELLOW >= 0.40, RED < 0.40
   - After each batch, calls `onProgress?.(completedCount, totalCount)`
   - Returns `ScoredClaim[]`
   - Include a local `chunk<T>(arr, size)` helper (do NOT import from lodash)

3. Export threshold constants: `GREEN_THRESHOLD = 0.65`, `YELLOW_THRESHOLD = 0.40`, `BATCH_SIZE = 10`

Import `Claim` from `./claim-extractor.js` and `SearchResult`, `EzSearchBridge` from `../ez-search-bridge.js`.

**report.ts:**

1. Export `DriftReport` interface:
```typescript
export interface DriftReport {
  sourceFile: string;
  healthScore: number;
  scoredClaims: ScoredClaim[];
}
```

2. Export `computeHealthScore(scoredClaims: ScoredClaim[]): number`:
   - If no claims, return 100
   - Otherwise: `Math.round(mean(scores) * 100)`

3. Export `buildDriftReport(sourceFile, scoredClaims): DriftReport` that creates the report object with computed health score.

4. Export `renderDriftReport(report: DriftReport): string` using string builder pattern (lines.push / join):
   - Header: `# Drift Report`, health score, file name, claim count
   - Group claims by status: Confirmed (GREEN), Possibly Stale (YELLOW), Contradicted (RED)
   - Each claim line: `- [STATUS] claim.text (score: X.XX)`
   - Under each RED/YELLOW claim, show top 2 evidence items: `  - file: chunk_preview` (truncate chunk to 80 chars)
   - Summary line at bottom with counts per status
