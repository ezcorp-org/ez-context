<task type="auto">
  <name>Task 1: Create claim extractor module</name>
  <files>src/core/drift/claim-extractor.ts</files>
  <action>
Create `src/core/drift/` directory and `claim-extractor.ts` with:

1. Export the `Claim` interface:
```typescript
export interface Claim {
  text: string;           // The claim text (bold/code markers stripped)
  sourceFile: string;     // Which file it came from
  sourceLine: number;     // 1-based line number
  sourceSection: string;  // Nearest parent heading
}
```

2. Export `extractClaims(content: string, sourceFile: string): Claim[]` that:
   - Splits content by newline, iterates with line tracking
   - Tracks current section from `#{1,3}` headings (store heading text, don't emit heading as claim)
   - Extracts bullet points matching `/^[-*+]\s+(.+)/`
   - Extracts numbered items matching `/^\d+\.\s+(.+)/`
   - Strips bold markers (`**text**` -> `text`) and inline code markers (`` `text` `` -> `text`) from extracted text
   - Skips lines containing `ez-context:` (marker comments)
   - Skips lines starting with `<!--` (HTML comments)
   - Skips blank/empty lines
   - Filters out claims shorter than 10 chars or longer than 300 chars
   - Filters out boilerplate value patterns matching `/^(Language|Framework|Build|Package Manager|Test Runner|Pattern|Layers):\s/i`

Refer to the research code example in `06-RESEARCH.md` Pattern 1 for the exact implementation. The SKIP_PATTERNS and BOILERPLATE_VALUE regexes are documented there.
  </action>
  <verify>Run `bun run typecheck` -- no type errors in the new file.</verify>
  <done>claim-extractor.ts exports Claim interface and extractClaims function that parses markdown into claims with section tracking and boilerplate filtering.</done>
</task>
