**types.ts** -- Add `OutputFormat` type and extend `EmitOptions`/`EmitResult`:
```typescript
export type OutputFormat = "claude" | "agents" | "cursor" | "copilot" | "skills" | "rulesync" | "ruler";
```
Add `formats?: OutputFormat[]` to `EmitOptions`. Change `EmitResult` to use a generic `rendered: Record<string, string>` map (keyed by format name) instead of fixed `claudeMd`/`agentsMd` fields. Keep `claudeMd` and `agentsMd` as convenience getters or computed properties for backward compatibility -- OR keep them as optional fields alongside `rendered`. Whichever is simpler. Also keep `filesWritten: string[]`.

**cursor-mdc.ts** -- Export `renderCursorMdc(registry, threshold): string`. Use `js-yaml` to serialize the YAML frontmatter block:
```
---
description: Project conventions extracted by ez-context
globs: ""
alwaysApply: true
---
```
Then render markdown body using the same `prepData` pattern from claude-md.ts (filter by threshold, group by category, render sections). Keep it concise -- conventions as bullet points, stack summary, key commands. The body should be a condensed version of CLAUDE.md content (Cursor rules have no specific structural requirements beyond the frontmatter).

**copilot-md.ts** -- Export `renderCopilotMd(registry, threshold): string`. Plain markdown, no YAML frontmatter. Similar content to CLAUDE.md but slightly restructured -- a single-file summary of conventions. Include a header comment `<!-- Generated by ez-context. Do not edit between markers. -->`.

**skill-md.ts** -- Export `renderSkillMd(registry, threshold): string`. YAML frontmatter with `name: "ez-context"` and `description` (max 1024 chars summarizing conventions). Body contains conventions organized for progressive disclosure: most important conventions first (highest confidence), then details. Keep total under 5000 tokens (~3750 words) as per SKILL.md spec.

**rulesync-md.ts** -- Export `renderRulesyncMd(registry, threshold): string`. YAML frontmatter with:
```yaml
targets:
  - cursor
  - copilot
  - windsurf
```
Body is convention markdown similar to copilot format.

**ruler-md.ts** -- Export `renderRulerMd(registry, threshold): string`. Plain markdown, no frontmatter. Direct convention dump -- simplest format. Header, stack, conventions, commands.

All emitters follow the same signature: `(registry: ConventionRegistry, threshold: number) => string`. Use the string-builder pattern (lines.push / join) established in claude-md.ts and agents-md.ts. Do NOT use Eta templates (decision from Phase 3). Import `js-yaml` for YAML serialization in cursor-mdc.ts, skill-md.ts, and rulesync-md.ts. Reuse the `prepData`-style filtering/grouping logic -- extract a shared helper if the pattern repeats more than twice.

**index.ts** -- Refactor `emit()` to support the `formats` field from EmitOptions:

1. Create a `FORMAT_EMITTER_MAP` that maps each OutputFormat to its `{ render, filename, writeStrategy }`:
   - `claude` -> `{ render: renderClaudeMd, filename: "CLAUDE.md", strategy: "markers" }`
   - `agents` -> `{ render: renderAgentsMd, filename: "AGENTS.md", strategy: "markers" }`
   - `cursor` -> `{ render: renderCursorMdc, filename: ".cursor/rules/ez-context.mdc", strategy: "direct" }`
   - `copilot` -> `{ render: renderCopilotMd, filename: ".github/copilot-instructions.md", strategy: "markers" }`
   - `skills` -> `{ render: renderSkillMd, filename: ".skills/ez-context/SKILL.md", strategy: "direct" }`
   - `rulesync` -> `{ render: renderRulesyncMd, filename: ".rulesync/rules/ez-context.md", strategy: "markers" }`
   - `ruler` -> `{ render: renderRulerMd, filename: ".ruler/ez-context.md", strategy: "direct" }`

2. Default `formats` to `["claude", "agents"]` when not specified (backward compat).

3. Loop over requested formats, render each, write using strategy (writeWithMarkers for "markers", writeFile for "direct"). For "direct" strategy, use `mkdir` with `{ recursive: true }` before writeFile to ensure parent dirs exist (e.g., `.cursor/rules/`).

4. Return `EmitResult` with rendered content map and filesWritten array. Maintain backward-compatible `claudeMd`/`agentsMd` fields (pull from rendered map, default to empty string if format not requested).

5. Re-export all render functions and the `OutputFormat` type.
