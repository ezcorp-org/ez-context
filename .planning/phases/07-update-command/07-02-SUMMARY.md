# Phase 7 Plan 02: Update Command CLI Wiring Summary

**One-liner:** Wire `ez-context update` CLI command using updateFile+extractConventions with --file/--dry-run/-y flags and 9 CLI tests.

## What Was Built

Completed the user-facing `ez-context update` command by:

1. Creating `src/commands/update.ts` — the `updateAction` handler that:
   - Checks for search index (exits 1 if missing)
   - Resolves target files: CLAUDE.md and AGENTS.md only (or --file override)
   - Calls `extractConventions` exactly once before looping files
   - Calls `updateFile(filePath, registry, bridge)` per file
   - Reports updated/skipped/aborted per file with chalk formatting
   - In --dry-run mode: scores claims and prints "Would update" / "Up to date" per file without writing

2. Updating `src/cli.ts` to register the `update` command with correct argument/option signatures.

3. Creating `test/commands/cli-update.test.ts` with 9 tests covering all branches.
