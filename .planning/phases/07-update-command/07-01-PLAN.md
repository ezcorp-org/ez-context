  <name>Task 1: Create src/core/updater.ts with validateMarkers, backupFile, and updateFile</name>
  <files>src/core/updater.ts</files>
  <action>
Create `src/core/updater.ts` with three exported functions and their types:

**Types:**
- `MarkerValidation` interface: `{ valid: boolean; mode: "create" | "append" | "splice" | "invalid"; reason?: string; startIdx?: number; endIdx?: number }`
- `UpdateAction` type: `"skipped" | "updated" | "aborted"`
- `FileUpdateResult` interface: `{ filePath: string; action: UpdateAction; reason: string; backupPath?: string }`

**`validateMarkers(content: string): MarkerValidation`**
- Import `MARKER_START`, `MARKER_END` from `../emitters/writer.js`
- No markers at all -> `{ valid: true, mode: "append" }`
- Both markers present, start < end -> `{ valid: true, mode: "splice", startIdx, endIdx }`
- Only one marker present (start without end OR end without start) -> `{ valid: false, mode: "invalid", reason: "Unpaired ez-context marker: {which} marker missing" }`
- End marker before start marker -> `{ valid: false, mode: "invalid", reason: "End marker appears before start marker (corrupted file)" }`
- CRITICAL: This is different from writeWithMarkers' silent guard. validateMarkers rejects unpaired markers; writeWithMarkers silently appends. The update command uses validateMarkers as a pre-flight check.

**`backupFile(filePath: string): Promise<string | null>`**
- If file doesn't exist (use `existsSync`), return `null`
- Copy file to `filePath + ".bak"` using `copyFile` from `node:fs/promises`
- Return the backup path string
- Overwrites existing `.bak` file silently (this is correct -- backup represents state before THIS run)

**`updateFile(filePath, registry, bridge, confidenceThreshold?): Promise<FileUpdateResult>`**
- Parameters: `filePath: string`, `registry: ConventionRegistry` (pre-computed, NOT extracted per-file), `bridge: EzSearchBridge`, `confidenceThreshold: number = 0.7`
- If file doesn't exist -> return `{ action: "skipped", reason: "File does not exist" }`
- Read file content with `readFile(filePath, "utf-8")`
- Validate markers with `validateMarkers(content)` -- if invalid, return `{ action: "aborted", reason: validation.reason }`
- If mode is "splice" (markers exist): extract claims with `extractClaims(content, filePath)`, score with `scoreClaims(claims, bridge)`. If NO drift (all GREEN), return `{ action: "skipped", reason: "No drift detected" }`. If claims array is empty, also skip (nothing to check).
- If mode is "append" (no markers): always proceed with update (file has no generated section yet)
- Create backup with `backupFile(filePath)` -- ONLY at this point (after validation and drift check pass)
- Determine emitter: `path.basename(filePath).toLowerCase().includes("agents")` -> `renderAgentsMd(registry, confidenceThreshold)`, otherwise -> `renderClaudeMd(registry, confidenceThreshold)`
- Write with `writeWithMarkers(filePath, newContent)`
- Return `{ action: "updated", reason: "Re-rendered drifted sections", backupPath }`

Import from existing modules:
- `MARKER_START`, `MARKER_END`, `writeWithMarkers` from `../emitters/writer.js`
- `renderClaudeMd` from `../emitters/claude-md.js`
- `renderAgentsMd` from `../emitters/agents-md.js`
- `extractClaims` from `./drift/claim-extractor.js`
- `scoreClaims` from `./drift/claim-scorer.js`
- `EzSearchBridge` type from `./ez-search-bridge.js`
- `ConventionRegistry` type from `./schema.js`
