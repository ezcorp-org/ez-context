**Create `src/commands/update.ts`** following the exact pattern of `drift.ts` and `generate.ts`:

```
export async function updateAction(
  pathArg: string,
  options: { file?: string; dryRun?: boolean; yes?: boolean }
): Promise<void>
```

Implementation:
1. `const projectPath = path.resolve(pathArg)`
2. Start spinner: `ora("Checking for drift...").start()`
3. `const bridge = await createBridge(projectPath)`
4. Guard: `if (!(await bridge.hasIndex(projectPath)))` -> spinner.fail("No search index found"), console.error with chalk.red suggesting to run generate first, `process.exit(1)`
5. Resolve target files:
   - If `options.file`: `[path.resolve(projectPath, options.file)]`
   - Otherwise: auto-detect only `CLAUDE.md` and `AGENTS.md` (NOT .cursorrules/CONTEXT.md -- update only ez-context-generated files). Filter with `existsSync`.
   - Use a `const UPDATE_CANDIDATES = ["CLAUDE.md", "AGENTS.md"]` constant.
6. If no files found -> spinner.fail("No context files found"), console.error, `process.exit(1)`
7. If NOT dry-run: call `extractConventions(projectPath)` ONCE to get registry. Update spinner text: "Extracting conventions..."
8. If dry-run: still need registry for preview rendering, so extract conventions too.
9. Loop over files, call `updateFile(filePath, registry, bridge)` for each.
   - Update spinner text per file: `"Updating {basename}..."`
10. Collect results. After loop:
    - If ALL skipped -> spinner.succeed("All context files are up to date"), exit 0
    - If any updated -> spinner.succeed with count of updated files
    - If any aborted -> report aborted files with chalk.yellow warning
11. Print per-file results:
    - "updated" -> chalk.green checkmark + filename + "(backup: filename.bak)"
    - "skipped" -> chalk.gray dash + filename + reason
    - "aborted" -> chalk.yellow warning + filename + reason
12. Wrap entire body in try/catch: spinner.fail("Update failed"), console.error(chalk.red(message)), process.exit(1)

**Dry-run mode (options.dryRun):**
- Extract conventions and check drift for each file, but do NOT call `updateFile`
- Instead, for each file: read content, call `validateMarkers`, extract claims, score claims
- Import `extractClaims` from `../core/drift/claim-extractor.js` and `scoreClaims` from `../core/drift/claim-scorer.js` for dry-run analysis
- Print: "Would update {filename}" or "Up to date: {filename}" per file
- Print the DRY RUN banner (same style as generate.ts)
- NO file writes, NO backup creation

**Update `src/cli.ts`:**
- Add import: `import { updateAction } from "./commands/update.js";`
- Add command registration after the drift command:
```
program
  .command("update")
  .description("Update drifted sections in context files, preserving manual edits")
  .argument("[path]", "project root to analyze", ".")
  .option("--file <contextFile>", "specific context file to update")
  .option("--dry-run", "preview changes without writing files")
  .option("-y, --yes", "non-interactive mode")
  .action(updateAction);
```
